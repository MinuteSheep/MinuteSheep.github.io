团队协作流程

  • 试图推送自己的修改 git push origin dev
  • 如果推送失败，则因为远程分支比本地分支新，需要合并分支 git pull
  • 如果提示no tracking information,则说明本地分支与远程分支没有对接，使用对接命令对接 git branch --set-upstream-to dev origin/dev
  • 进行分支合并，如何有冲突就解决冲突，之后提交再推送
